机器学习初感悟

发表于 2019-04-28 | 更新于 2019-04-29 | 分类于 机器学习

最近在小象学院买了一个课《Python人工智能》，课程涉及了一些基本的机器学习算法，也包含了几个机器学习的实战项目，虽然都比较基础，但是从中还是可以学到很多东西，今天天阴，正好就一起好好整理一下吧。
先整体归纳一下，在这门课程当中学到的东西，以及自己的一些感悟，再就每个不同的项目进行简单的分析。

总

  1. 机器学习的整个过程大致可以分为：数据收集、数据处理、构建模型、训练模型、预测结果。共五个部分。在这次课程的大作业中每一个项目都包含了除数据处理的其余四个过程。在真正完成一个项目时，由于对预测结果的追求，并不是说五个过程顺序走一趟就可以了，很可能是一个循环往复的过程，不断尝试、不断改进，直到最终模型的预测结果达到让人满意的程度。
  2. 数据处理需要把收集到的人能读懂的数据，通过一定的方法和规则，转换成能够在算法中运算的数字信息。一般来说都是对其进行分类编码。在这一部分还要做的就是特征的取舍，并不是所有的数据都与结果相关。此外，收集到的数据很有可能包含缺失值，对于缺失值，可以选择放弃或是填充，在包含缺失值样本极少时可以放弃不完整样本；又或者，有某一个特征包含大量的缺失值，也可以抛弃这个特征；而填充策略则有：向上填充、向下填充、均值填充、中位数填充等。在这一部分啊啊啊，还可以通过图像来快速判断特征之间，特征与标签之间的关系，为之后的模型建立作准备。总之呢，就是把手头的数据处理的能够直接上算法使用，而且还要对特征与预测结果的关系心中有数。
  3. 构建模型不是随便拿一个算法模型套上去就好了，而是要根据对数据的把握和对不同模型的理解，尽可能的选择适合的一个或者几个模型。在实际情况中，构建模型和训练模型往往是分不开的，因为大部分模型的都包含有超参数，为了选择最好的超参数，和最好的模型，当然就需要在数据集上进行多次训练最后取其中最好的结果啦。
  4. 如果已经得到了满意的模型，那么只要将数据处理的过程和训练好的模型保存起来就可以对新的数据进行预测。

分

一共五个大作业，分别为：IBM员工流失预测、贷款审批结果预测、员工流失预测进阶、蘑菇聚类分析、泰坦尼克幸存者预测。

  1. IBM员工流失预测
  2. 贷款审批结果预测
  3. 员工流失预测进阶
  4. 蘑菇聚类分析
  5. 泰坦尼克幸存者预测
# 总结
